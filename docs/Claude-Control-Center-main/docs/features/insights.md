# Usage Insights

The Insights page (`/insights`) provides a high-level, human-readable report of your Claude Code activity and behavioral patterns.

## The Insights Report
The dashboard parses the `report.html` file generated by Claude Code's own `/insights` command.

### Key Sections
- **Summary Stats**: High-level metrics like total sessions, active days, and project count.
- **Usage Charts**: Visualizations of your activity over the reporting period.
- **Feature Highlights**: Summaries of which Claude Code features you use most effectively (e.g., "Effective use of Plan Mode", "Strong test-driven workflow").
- **Behavioral Patterns**: Detailed analysis of your interaction style, including strengths and areas for improvement.
- **The "Horizon"**: Suggestions for more advanced Claude Code features or workflows you haven't explored yet, complete with example prompts.

## Generating Insights
The Insights page is a **viewer** for data generated by Claude Code. To see the latest insights:

1. Open a terminal and run `claude` (Claude Code).
2. Run the `/insights` slash command.
3. Once the report is generated, return to the Claude Control Center dashboard and refresh the Insights page.

## Technical Details
- **Parsing**: The backend uses `BeautifulSoup4` to surgically extract data from the `report.html` file located in `~/.claude/usage-data/`.
- **Location**: The report is read from `CLAUDE_DIR / "usage-data" / "report.html"`.
