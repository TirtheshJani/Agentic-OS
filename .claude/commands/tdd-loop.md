# Prompt

Read the provided issue document.

1. Write a Vitest test in the `dashboard/tests/` directory that describes the desired behavior. Do not write the implementation yet.
2. Run the test to ensure it fails.
3. Read the failure logs and implement the minimum code required to make the test pass.
4. Run the test suite again.
5. Once passing, pause and ask me if we should refactor.

Rules: follow `standards/agentic-workflow.md` (vertical slices, verifiable acceptance criteria) and the karpathy-guidelines skill (surgical minimal changes). Never claim done without showing the passing test output.
